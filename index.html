TVS component breakdown

C2 is the bottleneckSheets
C1 - Trace type component

Scores each trace based on what type it is. Withdrawals are the most valuable to clients because they show where funds leave an exchange, so they score highest.

Withdrawal = 3pts | Deposit = 2pts | Wdr PSP = 1pt

Q1 avg: 2.42/trace (gap to max: 0.58). Lifting WIS pushes this toward 3.0.

C2 - Discovery component

Scores each trace based on whether the entity has been traced before. Withdrawal retraces are most valuable because they add new outflow data to an already-known entity.

Wdr retrace = 3pts | First-time discovery = 2pts | Dep retrace = 1pt

Q1 avg: 1.96/trace (gap to max: 1.04). This is the bottleneck. 25% of Q1 traces were deposit retraces (1pt), dragging C2 well below 2.0.

Operational improvements

Shipping in Q2
NowPayments batch deposits

Deposit funding is now automated via NowPayments batch CSV uploads. The master operations sheet generates a batch file with amounts set 10% above minimum thresholds to absorb fees and slippage.

Impact: Eliminates manual wallet-by-wallet funding. A full batch of deposits submitted in one action. Reduces deposit turnaround from days to hours.

Status: Live on the master ops sheet.

QC-gated payment model

Replacing hourly pay with per-trace, QC-approved payments. Freelancers get paid only for traces that pass QC with complete metadata.

Rates: New entity deposit $5.00, additional deposit $3.50, withdrawal $4.00, withdrawal PSP $4.50, entity research (pre-approved) $1.50, metadata bonus $0.50/trace.

Impact: Aligns pay with output. Eliminates $38-45/trace outliers seen in Q1.

Status: Pending budget sign-off. Pilot planned April.

Sheet automations

Master ops sheet syncs status changes bidirectionally with freelancer sheets. Ingestion sweep runs every 6 hours across all freelancer sheets.

Fee tracker recovers blockchain dates and fees via block explorer APIs across 25+ chains.

Next: Weekly metrics auto-calculates TVS, WIS, traces per hour every Monday. QC quality tracking to be integrated once per-trace model is live.

Q2 outlook

Entity discovery deprioritised for Q2. Focus shifts to retracing already-traced entities with withdrawal-heavy batches. This directly addresses the C2 bottleneck: retrace work scores higher on C2 than first-time discovery, and when weighted toward withdrawals, lifts both C1 and C2.

QC-gated payments, once approved, will add a QC pass rate metric to future reports. Target: 90%+ first-time pass rate. This will appear as a new section in the Q2 dashboard.
